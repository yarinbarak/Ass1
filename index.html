Q & A

3 main differences between Web1, Web2 and Web3

1.

In web1 - Search could be done only by looking for the information in the html page (physically most of the times).

In contrast, Web2 allows search with search engines, and also supports semantics. Web3 allows natural language search in the queries (search engine) and machine learning is used to return answers.

2.

Pages in Web1 were static. If you would want to change something in the page you need to update the whole page - what means they weren't updated.
In contrast, Web2 supports sharing, updates are opptional all the time - what makes them more updated (frequently updated).
Web3 pages supposed to be updates all the time.

3.

Web1 pages didn't allow full interaction with the users - they couldn't share.
Web2 supports Sharing and collaboration.
Web3 uses the web to get information and by that allows users to be a part of information gathering and sharing.


Same query in Web1 and Web3.

Query: Sideaffects of Acamol

Web1

User had a group of web pages and the query that he wants to get an answer for.
The user is going over the pages he has (or knows about), checks the subject of the page. For each Page that is relevant to the query - medical, drugs - he searches the answer he needs in the page according to his query - looking for key words (physically), acamol and Sideaffects.
The user finds the Sideaffects of Acamol and go out of the web page.
The user can search in another page if he didnt find out what the Sideaffects are.
The user has the answer to the query but he spent much time and effort to look for it.

Web3

The user types the query to a search engine and it returns the best suiting results for his query.
The user go over the results and find out what the Sideaffects are.
He doesnt even need to look inside the pages, only on the answers that came back from the search. (We learned that web3 returns specific answers and their locations in pages.)
The user has the answer to his query, he didn't have to spend too much time or effort.
